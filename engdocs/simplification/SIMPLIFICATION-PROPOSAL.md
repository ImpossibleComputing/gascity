# Reconciler-Cluster Simplification Proposal

_Generated 2026-07-07 · Fable discovery (11 subsystem maps, 66 candidates) → dedup → elaborate → rank. Base `f6a0fbd5f`, branch `simplify/audit-2026-07-07`._

## Headline

The cluster's core disease is edge-triggered thinking bolted onto a system whose reliability story is level-triggered convergence: instead of one idempotent per-tick converge pass over persistent state, the reconcilers accreted event-shaped special cases, staleness defenses, migration scans, and triplicated copies of the same merge/attach/close logic — and most of the 100+ tracked bugs live in the gaps between those copies. The second axis is durable-vs-runtime divergence: the bead ledger and the live session/process world are reconciled ad hoc at a dozen sites (pool demand, pendingCreate leases, drift relaunch, ghost sessions) rather than through one Observe→Plan→Materialize pipeline with typed state. The winning sequence is: codify the convergence blueprint (S35), land the cheap error-surfacing and typing guardrails, then collapse the reconciler onto a single level-triggered converge loop while deleting the legacy dual-mode machinery it obsoletes.

**38 items** · 20 auto-land · 18 needs-julian · 81 candidate branches.

## Cross-cutting themes

### Edge-triggered to level-triggered convergence

Replace event-shaped special cases and one-shot handlers with idempotent per-tick converge passes over persistent state: the system already converges because work persists, so make the code shape match — one converge loop for sessions, a sweep for ownerless coordination beads, pool satisfaction as a level with hysteresis, scope rebuild on reload, and a commit-point helper that makes write-ordering violations unrepresentable.

Items: `S19`, `S22`, `S24`, `S21`, `S34`, `S30`, `S35`

### Durable-vs-runtime as one reconciled truth

The bead ledger and the live runtime diverge at every site that improvises its own comparison. Pure Observe/Plan/Materialize desired-state construction, a typed PendingCreateLease, drift-relaunch routed through the one launch-config derivation, and a ghost-session healer collapse the divergence family into a single reconciliation contract.

Items: `S24`, `S28`, `S36`, `S10`, `S08`, `S19`

### Type the wire: kill stringly state and metadata

Stringly session states, route strings, Info key soup, and ref-string surgery are where silent skips and misroutes breed. Typed SessionState with mandatory handlers, a typed Router claimed by kind, a table-driven Info codec, typed trace surface, one legacy-state canonicalizer, and durable lineage metadata turn whole bug classes into compile errors.

Items: `S20`, `S15`, `S09`, `S26`, `S38`, `S12`

### One implementation per concept (drift-by-duplication)

Triplicated attach pipelines, five normalization sites, six child-projection loops, three scope-close copies, dual attempt/create paths, and dual launch-guard/compile sites each drift independently — including at least one live behavior bug (trigger-config loss). Merging each to a single test-guarded implementation is the highest-confidence bug kill per line deleted.

Items: `S13`, `S18`, `S31`, `S37`, `S32`, `S05`, `S14`, `S01`, `S12`, `S36`

### Finish migrations, then split god files on their seams

Half-finished fronts (Info front-door, worker boundary idioms) force every reader to handle both worlds. Complete the migration first — one tick-scoped mutator, delete the ~26 raw/Info pairs, collapse the Create-variant telescope and config accessor boilerplate — and the 4,900-LOC files then split cleanly along already-visible phase seams.

Items: `S23`, `S25`, `S11`, `S04`, `S06`

### Wire-or-delete: retire dead and legacy dual-mode machinery

Phantom half-built engines (wake, trace WAL, lifecycle fold remnants), per-tick migration scans, store==nil mode, the v1 control engine, and the #2403 staleness taxonomy are pure carrying cost that also masks real signals. Deleting them shrinks the surface the convergence rewrite must reason about — and S35 records the negative result so nobody re-unifies what should stay separate.

Items: `S08`, `S29`, `S27`, `S34`, `S07`, `S35`

### Stop swallowing errors on destructive paths

Seven swallowed errors on convoy recovery, orphan close, and control routing convert recoverable faults into silent state corruption diagnosed hours later as 'molecule stalls'. Surfacing them is the single best win/(risk*effort) item in the set and should land first.

Items: `S16`, `S33`, `S02`, `S03`

### S16 — Surface the seven swallowed errors on destructive/routing paths (convoy recovery, orphan close, control routing) and cache the per-bead city.toml re-parse

*internal/sling (sling_attachment.go), cmd/gc session reconciler (session_reconciler.go), internal/dispatch (control.go, drain.go, ralph.go)* · risk **low** · effort **S** · **auto-land** · priority 1 · bugs #1053, #2987, #3872, #1294, #3892

**Problem.** Seven silent-degradation sites, all violating the project's Don't-Swallow-Errors rule, several on DESTRUCTIVE paths. (1) needsConvoyRecovery (sling_attachment.go:340-343) treats a parent-read error as 'recovery needed' and hasLiveTrackingConvoy (:358-365) treats a listing error as 'no live convoy' — a transient store hiccup makes finalize re-run and mint ANOTHER auto-convoy (sling_core.go:583), silent duplicate accumulation under exactly the degraded-store conditions of #2987. (2) session_reconciler.go:1538-1541 maps ANY workerSessionTargetRunningWithConfig error to providerAlive=false with no log, and !providerAlive authorizes the orphan bead CLOSE at :1943 — every neighboring guard fails safe on error (drain-ack :1794, max-age :2882, progress-stall :2227) but this one fails destructive: a tmux blip can close a bead whose runtime is alive (#3872-family divergence). (3) loadAttemptRouteConfig (control.go:963-972) swallows the city.toml load error, silently flipping attempt routing into the metadata-only fallback (:1002-1007) that may never be claimed (quiet cousin of #1938) — and it is a full TOML disk parse PER PROCESSED BEAD via beadUsesMetadataPoolRoute (:1150-1152) and retryPreservedAssignee (ralph.go:697-699). (4) mustReloadDrain (drain.go:1471-1477) returns the STALE bead on reload failure so completeDrain can run against a pre-transition drain state. (5) appendAttemptLogValue (control.go:1490-1491) ignores json.Unmarshal failure, silently resetting the gc.attempt_log audit history. (6) markControllerSpawnError (control.go:323, 330-331) discards SetMetadataBatch/setOutcomeAndClose errors on the error-RECORDING path.

**Proposal.** Bring every site to the codebase's existing fail-safe convention. Sling: thread errors out of needsConvoyRecovery/hasLiveTrackingConvoy (or fail CLOSED: on error assume the convoy exists, report Idempotent with a warning in BeadCheckResult.Warnings — the field already exists). Reconciler: log the liveness error and treat it as 'observation unavailable' — skip the destructive close this tick (level-triggered loop retries next tick), or at minimum gate the close (not the drain) on err==nil. Dispatch: give ProcessOptions a lazily-cached routeConfig() (*config.City, error) resolved once per ProcessControl invocation and propagate the load error to routing decisions; mustReloadDrain -> reloadDrain returning error (its caller at drain.go:194 already returns errors); trace corrupt attempt logs; trace ignored writes in markControllerSpawnError.

**Wins.**
- _maintainability_: All error handling in the reconciler ladder and control engine converges on one stated convention (fail safe, trace, retry next tick), matching the already-correct sites (artifact purge runtime.go:767-771).
- _testability_: Each failure mode becomes testable with an erroring fake store — today 'store error' is indistinguishable from 'no convoy' / 'dead session' / 'no config', so the paths are untestable-by-inattention.
- _correctness_: Closes three silent-damage vectors: duplicate auto-convoy minting, false orphan-close of live-runtime beads, and silent routing degradation to never-claimed metadata routes; converts audit-log resets and lost error-recording writes into visible traces. Kills the swallowed-error-on-destructive-path class in these two subsystems, not one instance.
- _complexity_: ~15 LOC (sling) + 3-5 LOC (reconciler) + ~net-zero LOC (dispatch); city.toml parses drop from up-to-N-per-cycle to 1 per ProcessControl invocation (measurable on busy dispatchers, complements #3892).

### S20 — Typed SessionState enum with mandatory handlers, plus an escape hatch and throttle for the unknown-state skip

*Session reconciler core (cmd/gc/session_reconciler.go, cmd/gc/session_reconcile.go, cmd/gc/cmd_start.go) + internal/session* · risk **low** · effort **M** · **auto-land** · priority 2 · bugs #1497, #2085, #2389, #3279, #1542

**Problem.** Session state is a raw metadata string and the reconciler's only defense against an unrecognized value is skip-and-continue: session_reconciler.go:1523-1532 logs 'unknown state' to stderr and continues — skipped forever with no first-seen record, no TTL, no escalation, and the line re-logged every tick per session. This one site IS #1497 (no escape from the skip), #2085 (gc restart doesn't recover — the state is durable and the skip unconditional), and #2389 (per-tick log spam). ~10 stringly state predicates (pendingCreateQueuedOrCreatingState :976, pendingCreateRollbackState :985, isKnownStateInfo, ...) scatter the state taxonomy, part of 29 raw Metadata keys at ~252 access sites across the four reconciler files. Relatedly, rate-limit quarantine (session_reconcile.go:753-763) uses a fixed duration with zero jitter on restart paths (only max_session_age has jitter, cmd_start.go:309), producing the #3279 synchronized 429 respawn storm.

**Proposal.** Two staged slices. Slice 1 (signal, ~40 LOC): keep the forward-compat skip but add (a) a once-per-session-generation durable unknown_state_first_seen marker via the existing sessFront patch machinery, (b) one typed event registered via events.RegisterPayload, emitted on first observation and again past a threshold age, and (c) transition-triggered logging (first sight / state change only), reusing the exact throttle-marker pattern already used by emitSessionStrandedDiagnostic (:3336). Slice 2 (enum): introduce a typed SessionState enum in internal/session with an exhaustive switch in the reconciler (exhaustive lint makes a missing handler a build error) and one explicit Unrecognized{firstSeen, raw} state carrying a TTL policy; attach per-state backoff policy (duration + jitter + optional Retry-After) to the enum so THROTTLED gets jittered wakeup for free, killing the synchronized respawn storm. Escalation past TTL is a signal (typed event / queryable marker), not an auto-mutation — no decision logic in Go; the forward-compat rollback window stays intact because Unrecognized still defers action.

**Wins.**
- _maintainability_: An invisible stall becomes a queryable bead marker + typed event; state semantics live in one policy table instead of scattered predicates; adding a state forces handling everywhere via the compiler.
- _testability_: State policy (backoff, TTL, escalation) becomes a pure table with table-driven tests; the throttle logic reuses an already-tested pattern.
- _correctness_: Makes 'a state nobody handles' unrepresentable at compile time; closes #2389 fully and the observability half of #1497/#2085 in slice 1, the convergence-pressure half via TTL escalation in slice 2; jittered per-state backoff kills the #3279 storm class, not one instance.
- _complexity_: Replaces ~10 stringly state predicates with one enum + policy table; starts shrinking the 29 raw Metadata keys / ~252 access sites; stringly→typed is exactly the flagged auto-land category.

**Competing approaches.**
- **a** `simplify/s20-a` — Signal-only slice: durable unknown_state_first_seen marker + typed event (RegisterPayload) + transition-triggered logging via the existing emitSessionStrandedDiagnostic throttle-marker pattern. Recovery stays with operators/agents. ~40 LOC, one new event constant. _Tradeoffs:_ Safest, fully consistent with keep-judgment-out-of-Go, overnight-safe. Adds LOC rather than deleting — earns its place purely via the #1497/#2085/#2389 cluster. Does not close #3279 or the stringly-predicate sprawl. _Files:_ cmd/gc/session_reconciler.go, internal/events/registry.go (event constant + payload), cmd/gc/session_reconciler_test.go
- **b** `simplify/s20-b` — Full typed enum: SessionState in internal/session, exhaustive switch in the reconciler, Unrecognized{firstSeen, raw} with TTL escalation-as-signal, per-state backoff policy table with jitter (THROTTLED gets jittered wakeup, fixing #3279). Replaces the ~10 stringly predicates. _Tradeoffs:_ The real structural win — compile-time exhaustiveness plus the policy table — at the cost of touching classification sites across four reconciler files. If TTL escalation is ever upgraded from signal to auto-heal (heal-to-asleep on NOT-alive sessions, the healStatePatchWithRollback shape at session_reconcile.go:1195), that weakens the version-rollback safety window and needs Julian's sign-off on TTL semantics; the signal-only escalation shipped here does not. _Files:_ internal/session/state.go (new), cmd/gc/session_reconciler.go, cmd/gc/session_reconcile.go, cmd/gc/cmd_start.go, internal/events/registry.go, internal/session/state_test.go
- **c** `simplify/s20-c` — Staged a-then-b on one branch lineage: land the signal slice immediately, then rebase the enum work on top with field data from the new event informing the TTL default and the escalation threshold. _Tradeoffs:_ Best sequencing (observability before policy) and each stage independently revertible; slightly more process overhead than landing b directly, and the marker key must be chosen so the enum slice can adopt it unchanged. _Files:_ cmd/gc/session_reconciler.go, internal/session/state.go (new, stage 2), cmd/gc/session_reconcile.go, internal/events/registry.go

### S24 — Pool satisfaction as a level (bead-open AND runtime-alive, with hysteresis) plus a ghost-session ledger healer

*Desired-state construction + pool demand (cmd/gc/build_desired_state.go, cmd/gc/pool_desired_state.go, cmd/gc/session_lifecycle_parallel.go, cmd/gc/city_runtime.go)* · risk **medium** · effort **M** · **needs-julian** · priority 4 · bugs #1029, #2083, #2285, #3554, #3872, #1542

**Problem.** Demand is derived from a single-instant snapshot with no runtime-liveness input, wrong in both directions. Under-count: collectAssignedWorkBeadsWithStores (build_desired_state.go:716-757) collects assigned-work beads once per tick and filterAssignedWorkBeadsForPoolDemand (build_desired_state.go:844) computes demand from it — a session whose assignedWorkBeads is briefly zero during a step transition looks idle and gets reaped mid-work (#1029). Over-count: computePoolDesiredStates (pool_desired_state.go:108-145) consumes durable sessionInfos only, so a pool bead whose tmux runtime died but whose bead stayed open counts as staffed indefinitely — #3872 incident 4 (33 READY beads, zero workers, no replacements); same family as #1542 stuck-creating. Multiple deciders: gc sling spawns a pool session that a subsequent gc session new --alias duplicates (#2083) because two writers re-derive demand independently. And rig-keyed store lookups (build_desired_state.go:549 rigStores[rig.Name]) go silently empty after rig remove + adopt re-add (#2285) with no divergence signal.

**Proposal.** Adopt the #3872 authors' own rule as ONE satisfaction predicate: a pool slot is SATISFIED iff session-bead-open AND runtime-alive, and a session is REAPABLE only after K consecutive ticks of no-open-work (hysteresis over the level, not a reaction to one edge). Put the predicate in one function that both the reconciler and sling consult, eliminating the second demand decider. Do NOT thread liveness checks into every demand reader (that duplicates state); instead add ONE level-triggered healer in the per-tick reconcile: for open pool/named session beads in state active/awake whose runtime is observably absent — reusing poolSessionBeadRuntimeRunningInfo (city_runtime.go:2820) and the existing liveness seam observeRuntimeProviderLiveness (session_lifecycle_parallel.go:1749) — transition the bead to the existing orphaned/asleep state after a 2-consecutive-tick grace window (guarding the #1029 transient-zero race). Demand math stays purely durable; the ledger is forced to converge to runtime truth within a bounded number of ticks. Make an empty rigStores hit for a configured rig a surfaced diagnostic instead of a silent skip (#2285).

**Wins.**
- _maintainability_: Matches the documented convergence philosophy ('the system converges because work persists'); demand remains a pure function of beads, keeping the model explainable.
- _testability_: Hysteresis is a pure function of a tick series; the healer is a pure predicate over (Info, liveness observation, tick count) — unit-testable with runtime.Fake.
- _correctness_: Closes #1029 (transition blips no longer reap) and the #3872 ghost-staffed starvation class at the source; removes the #2083 duplicate-spawn race at its root (one decider); surfaces the #2285 silent skip; #1542-shape stuck-creating is already covered by the pendingCreate lease sweep — this covers the active-state gap.
- _complexity_: One satisfaction predicate instead of per-caller re-derivation; one convergence rule instead of N alive-checks sprinkled through demand/suppression/scale-check readers.

**Competing approaches.**
- **a** `simplify/s24-a` — Recommended: shared satisfaction predicate + K-tick reap hysteresis + ledger-healer with 2-tick debounce transitioning runtime-dead open beads to orphaned/asleep; demand math stays purely durable. _Tradeoffs:_ Risk is a false positive when a provider's ListRunning/ProcessAlive is transiently wrong (tmux server restart) — mitigated by the debounce plus the existing IsPartialListError guard adoption already uses (adoption_barrier.go:77). Convergence is bounded but not instant. _Files:_ cmd/gc/build_desired_state.go, cmd/gc/pool_desired_state.go, cmd/gc/session_lifecycle_parallel.go, cmd/gc/city_runtime.go, cmd/gc/session_reconciler.go

### S02 — Per-bead dirty overlay: stop declining the entire cache when one bead is dirty

*internal/beads (caching_store reads/handles — dirty-set semantics)* · risk **medium** · effort **M** · **needs-julian** · priority 31 · bugs #2987, #2153

**Proposal.** Treat dirty as a per-bead overlay, not a global tripwire: cached read paths refresh only the dirty IDs via bounded backing.Get (|dirty| round-trips, typically 1-2) and serve everything else from cache; a successful Get absorbs the row (reusing S01's absorbFreshLocked) and clears the flag. **Wins.**
- _maintainability_: Dirty semantics defined in one place instead of six read-site guards; net LOC roughly flat to slightly negative but the decline-everything special case disappears.
- _testability_: Overlay refresh is a bounded, deterministic per-ID path unit-testable with a fake backing; today's behavior is only observable via full-scan fallbacks under load.
- _correctness_: Breaks the #2987 amplification loop; dirty-row staleness window drops from next-full-reconcile (30-120s+) to <=5s or read-time; unverifiable foreign events (#2153) converge without waiting for full reconcile.
- _complexity_: One overlay rule replaces the tri-condition duplicated at 6 read sites; efficiency: one dirty bead costs 1 backing.Get instead of an O(active-set) backing List per read.

Recommended first slice. _Tradeoffs:_ Safest: preserves both the eventual-consistency contract and the existing escape hatch exactly; slightly more code than (a) (cap constant + fallback branch), and pathological dirty growth still degrades to today's behavior rather than fixing it. _Files:_ internal/beads/caching_store_reads.go, internal/beads/caching_store_handles.go, internal/beads/caching_store.go, internal/beads/caching_store_internal_test.go
